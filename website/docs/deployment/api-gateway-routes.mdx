---
title: Configuring Routes in API Gateway
description: Set up a catch-all proxy route in API Gateway so every request reaches your Lambda API function.
---

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

Routes must be configured in API Gateway in order to support routing to the Lambda function. The easiest way to support all of your routes without recreating them is to use [API Gateway's Proxy Integration](https://docs.aws.amazon.com/apigateway/latest/developerguide/api-gateway-set-up-simple-proxy.html#api-gateway-proxy-resource?icmpid=docs_apigateway_console).

Simply create a `{proxy+}` route that uses the `ANY` method and all requests will be routed to your Lambda function and processed by the `lambda-api` module. In order for a "root" path mapping to work, you also need to create an `ANY` route for `/`.

<Tabs groupId="apigw">
  <TabItem value="rest" label="REST API (v1)">

For a REST API, create a greedy proxy resource `{proxy+}` with the `ANY` method, using Proxy Integration to forward requests to your Lambda function. To also handle the root path, add an `ANY` mapping for `/`. Together these forward every request to your function so Lambda API can route it.

  </TabItem>
  <TabItem value="http" label="HTTP API (v2)">

For an HTTP API, use a catch-all `$default` route (equivalent to `ANY /{proxy+}`) to forward all requests to your Lambda function. Lambda API automatically detects the v2 payload format, so no code changes are needed.

  </TabItem>
</Tabs>
