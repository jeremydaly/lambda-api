---
title: The RESPONSE Object
description: Overview of the chainable RESPONSE object used to send responses back to API Gateway.
---

The `RESPONSE` object is used to send a response back to the API Gateway. The `RESPONSE` object contains several methods to manipulate responses. All methods are chainable unless they trigger a response.

## Response methods by category

- [Sending Responses](/docs/request-response/sending-responses) — set status codes and send bodies with `status`, `sendStatus`, `send`, `json`, `jsonp`, `html`, and `type`.
- [Headers & Cookies](/docs/request-response/headers-and-cookies) — read and write response headers and manage cookies with `header`, `getHeader`, `getHeaders`, `hasHeader`, `removeHeader`, `getLink`, `cookie`, and `clearCookie`.
- [Location, Redirects & CORS](/docs/request-response/redirects-and-cors) — control navigation and cross-origin access with `location`, `redirect`, `cors`, and `error`.
- [Caching & ETags](/docs/request-response/caching-and-etags) — manage client caching with `etag`, `cache`, and `modified`.
- [Files & Downloads](/docs/request-response/files-and-downloads) — serve and download files with `attachment`, `download`, and `sendFile`.
- [Response Methods API Reference](/docs/api-reference/response-methods) — the full reference table of every RESPONSE method.
